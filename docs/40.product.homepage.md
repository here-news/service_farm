## 1. The Core Insight

**Traditional news homepages are graveyards.** A list of headlines sorted by recency or editorial whim. Static thumbnails. Click counts masquerading as importance.

**Our homepage is an observatory.** Users look into a living ecosystem where:
- Events have **mass** (importance derived from coverage depth)
- Events have **state** (ACTIVE/STABLE - are they breathing?)
- Events have **tension** (contradictions = unresolved debates)
- Events have **needs** (coverage gaps, unanswered questions)

Users don't just consumeâ€”they see where they can **contribute**.

## 4. Key Components

### 4.1 Live Ecosystem Visualization

The central visual showing events as organisms:

```typescript
interface EventOrganism {
  id: string;
  headline: string;
  mass: number;           // Determines size
  state: 'ACTIVE' | 'WARM' | 'STABLE' | 'DORMANT';
  tension: number;        // 0-1, affects color
  coherence: number;      // Health indicator
  claim_count: number;
  source_count: number;
  needs: string[];        // Coverage gaps
  last_activity: Date;
}

// Visual mapping
function getVisualProps(event: EventOrganism) {
  return {
    size: Math.sqrt(event.mass) * 20,  // Size scales with sqrt of mass
    animation: event.state === 'ACTIVE' ? 'pulse 2s infinite' : 'none',
    color: tensionToColor(event.tension),
    opacity: event.state === 'DORMANT' ? 0.5 : 1,
  };
}

function tensionToColor(tension: number): string {
  if (tension < 0.1) return '#4CAF50';  // Green - stable
  if (tension < 0.2) return '#8BC34A';  // Light green
  if (tension < 0.3) return '#FFC107';  // Yellow - some debate
  if (tension < 0.4) return '#FF9800';  // Orange
  return '#F44336';                      // Red - high tension
}
```

### 7.3 Event Ranking

```python
def rank_events_for_homepage(events: List[EventfulUnit]) -> List[EventfulUnit]:
    """
    Rank events for homepage display.
    Balance: importance (mass) Ã— activity (state) Ã— need (gaps)
    """
    def score(eu):
        mass_score = math.log(eu.mass + 1)  # Log scale for mass
        activity_score = {
            'ACTIVE': 3.0,
            'WARM': 2.0,
            'STABLE': 1.0,
            'DORMANT': 0.5
        }[eu.state]
        need_score = len(eu.coverage_gaps) * 0.5  # Events with needs rank higher
        recency_score = 1.0 / (1 + hours_since_activity(eu))

        return mass_score * activity_score * (1 + need_score) * recency_score

    return sorted(events, key=score, reverse=True)
```
